# Stage Snapshot (Completed Work)

This document records what has already been implemented in the current codebase.

## 1. PostgreSQL-Only Runtime

- Runtime database is PostgreSQL-only.
- `DATABASE_URL` default now points to PostgreSQL.
- `TEST_DATABASE_URL` is supported for test isolation.
- Non-PostgreSQL runtime is rejected at startup.
- Alembic migration environment enforces PostgreSQL dialect.

## 2. Migration Baseline Reset and Schema Evolution

- Active migration chain was reset to PostgreSQL-first baseline:
  - `0001_postgres_first_baseline`
- Follow-up migration for source name uniqueness:
  - `0002_source_name_uniqueness`
- Legacy SQLite-era migrations are archived under `docs/legacy_migrations/` and are non-executable.

## 3. Schema Guard and Failure Mode Hardening

- Added schema readiness guard (`app/db/schema_guard.py`).
- Startup checks Alembic current revision against head revision.
- DB-backed request paths return controlled `503` for schema mismatch/stale schema instead of raw SQL traceback `500`.
- Scheduler startup is skipped when schema is not ready.

## 4. Source Creation and Replacement Rules

- Source `name` is required (non-blank after trim).
- Added normalized uniqueness (`trim + lower`) with DB unique constraint on `sources.normalized_name`.
- Duplicate-name create now performs in-place replacement instead of creating a second source:
  - Same `source_id` is reused.
  - Source config fields are updated.
  - Historical source-scoped data is cleared (`events`, `snapshots`, `changes`, `notifications` via cascade, overrides).
- Create API now returns:
  - `was_replaced=false` on new create (`201`)
  - `was_replaced=true` on replacement (`200`)

## 5. Baseline-First Diff Semantics

- First successful sync per source is treated as baseline:
  - snapshots/events are written
  - no `changes` rows are created
  - no notification email is sent
  - response includes `is_baseline_sync=true`
- Subsequent syncs resume normal diff behavior.
- Removed-event detection keeps debounce rule (requires recent multi-snapshot absence before `removed` diff).

## 6. Diff Review UX and Evidence Download

- Diff card interaction changed to immediate action:
  - `Mark Viewed` / `Mark Unread` toggle submits instantly.
  - In `Unread` filter, viewed items disappear after successful update + refresh.
- Added evidence download endpoints:
  - `GET /v1/changes/{change_id}/evidence/before/download`
  - `GET /v1/changes/{change_id}/evidence/after/download`
- Evidence download is API-key protected and streams `.ics` attachment.
- Added safe evidence path resolver with root-boundary checks to block path traversal.

## 7. Frontend Behavior Updates

- Source form handles replacement response and shows replacement message.
- On replacement, UI asks whether to run immediate sync (baseline sync path).
- Course selector display format no longer shows redundant `raw -> raw` labels.
- Source refresh behavior remains read-only (does not trigger sync).

## 8. Dev Scripts and Docs

- Added PostgreSQL reset script: `scripts/reset_postgres_db.sh` (drop/create target DB + migrate to head).
- Startup script now waits for DB readiness, applies migrations, then starts server.
- README was simplified to PostgreSQL local flow (start Postgres, reset DB, start server).
- Architecture docs updated to PostgreSQL-only framing.
